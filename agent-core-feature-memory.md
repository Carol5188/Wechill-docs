## 1. 红包返奖玩法：必须记住的业务口径

### 1.1 基础规则

- 红包入口展示门槛：财富等级 10 级以上。
- 已上锁房间不可发送红包。
- 红包领取倒计时：3 分钟。
- 红包类型：金币红包、礼物红包。
- 红包个数不能为 1。
- 点击红包全服飘屏可以跳转到对应红包房间。
- 金币红包领取人获得金币；礼物红包领取人获得礼物或等价钻石；发送方返奖始终以金币结算。

### 1.2 返奖分档

金币红包：

| 总金额 | 返奖资格 | 概率 | 比例 |
| --- | --- | --- | --- |
| 3,000 | 否 | 0 | 0 |
| 5,000 | 否 | 0 | 0 |
| 10,000 | 否 | 0 | 0 |
| 20,000 | A 档 | 3% | 5% |
| 50,000 | B 档 | 5% | 10% |

礼物红包按礼物总价值折算：

| 礼物总价值 | 返奖资格 | 概率 | 比例 |
| --- | --- | --- | --- |
| < 20,000 | 否 | 0 | 0 |
| 20,000 - 49,999 | A 档 | 3% | 5% |
| >= 50,000 | B 档 | 5% | 10% |

公式：

- 礼物总价值 = 礼物单价 * 礼物数量。
- 返奖金额 = 红包总金额 * 返奖比例。
- 金币值向下取整。
- A 档期望补贴率 = 3% * 5% = 0.15%。
- B 档期望补贴率 = 5% * 10% = 0.50%。

### 1.3 开奖与不开奖条件

具备返奖资格的前提：

- 红包总金额或礼物总价值满足 A/B 档门槛。
- 红包个数 > 1。
- 红包发送成功。

真正触发抽奖必须同时满足：

- 红包在 3 分钟倒计时内被领完。
- 有效领取人数 >= 2。
- 发送人自领不计入有效领取人数。
- 发送人未处于限制级或危险级红包黑名单。
- 未命中风控规则。
- 未触发平台日返奖预算熔断。

结算时机：

- 红包领完后立即开奖并返还。
- 红包 ID 是唯一开奖主键。
- 开奖结果必须落库，重试不重复开奖。
- 发奖失败进入补发队列；补发成功后写钱包事务 ID，禁止二次补发。

不触发返奖的场景：

- 红包未领完或过期未领完。
- 有效领取人数不足 2。
- 命中风控拦截。
- 发送人处于限制级或危险级黑名单。
- 平台每日返奖预算触顶。
- 发奖失败且补发未成功。

## 2. 红包风控：必须记住的规则

### 2.1 黑名单等级

| 等级 | 用户体验 | 返奖影响 | 典型场景 |
| --- | --- | --- | --- |
| 危险级 | 不展示红包入口，禁发禁领 | 无返奖资格 | 明确作弊、脚本抢红包、生产模拟器套利、团伙主号 |
| 限制级 | 可发可领 | 发出的达标红包不参与概率返奖 | 关联设备/IP、高风险房间发送人、高频大额用户 |
| 观察名单 | 正常发领 | 暂不拦截，只监控和预警 | 证据较轻、待观察、B 档聚集观察 |

黑名单变更必须记录：操作人、来源、原因、有效期、关联设备/IP 快照。

### 2.2 风控规则和阈值

- 同设备 / 同 IP 多账号互抢：同一红包有效领取用户中出现多个同设备或高风险同 IP 账号，或发送人与固定一组账号高频互抢。处理为该红包不参与返奖、记录日志、进审核队列、关联账号同步限制。
- 高频大额红包：单用户 1 小时内达标红包次数 > 30，或达标红包总金额 > 3,000,000 金币。强风险进入限制级，并发飞书预警。
- B 档红包房间聚集：同房间 10 分钟内 >= 10 笔 B 档红包进入观察名单，只告警；>= 20 笔且自然互动弱、领取热度异常或关联账号重复时，判为强风险。
- 高风险房间：长期固定少量用户互抢、红包领完率异常高但自然礼物行为极低、中奖率显著异常。
- 日返奖上限：全站每日返奖预算 2,000,000 金币，约 200 美金。触顶后新领完红包记为 `blocked_by_budget`。
- 生产环境模拟器限制：模拟器、压测工具、内部测试账号、灰度验证账号不得参与真实返奖；命中模拟器默认危险级。
- 防脚本抢红包：服务端随机延迟窗口 1 - 3 秒，返回 `claim_not_before`；领取接口必须校验时间窗口和一次性行为 token。
- 领取响应监控：红包发出到领取 < 500ms 标记可疑；连续 N 个红包领取响应 < 1s 触发验证码或临时冷却，默认 N=3。

### 2.3 风控状态流转

建议状态包括：

- `normal`
- `waiting_draw`
- `blocked_by_risk`
- `blocked_by_blacklist`
- `blocked_by_budget`
- `risk_observing`
- `draw_lost`
- `draw_won`
- `reward_sent`
- `reward_retry`
- `reward_compensated`

## 3. 红包交互策略：必须记住的体验规则

- 发送方红包结果弹窗与返奖成功弹窗必须优先合并，避免连续打断。
- 同一红包同一角色只保留一个结果态，例如“红包最终结果卡”。
- 自己抢自己红包，且抢到最后一份时，也只展示一个合并结果弹窗。
- 当前已有高优先级弹窗时，返奖结果先降级为 toast、红点或消息中心，待强弹窗关闭后再补展示。
- 用户离开房间后，不强弹模态窗，改用系统消息、钱包到账通知、红包记录页红点。
- 多红包场景只保留一个主悬浮入口，徽标显示待领取红包数量，倒计时显示最早过期红包。
- 待领取红包数量 > 1 时，点击先进入红包列表底部弹层，而不是连续弹单个红包。
- 红包列表排序建议：即将过期优先、返奖资格优先、高金额优先、最新发送优先。
- 模态弹窗排队上限：3。超过后低优先级弹窗降级为 toast、列表状态更新或消息中心记录。
- 短时间多个红包结束时，可展示结果摘要卡，例如“你有 3 个红包已完成，1 个中奖返奖，2 个未中奖”。

弹窗优先级：

| 级别 | 内容 |
| --- | --- |
| S | 充值阻断、离房确认、安全风控强提示 |
| A | 返奖中奖结果、红包领完结果 |
| B | 红包领取弹窗、多红包选择弹窗 |
| C | toast、轻提示条、红点提醒 |

## 4. 红包后台信息架构

红包返奖后台建议一级菜单：`红包返奖`。

二级菜单：

- 基础配置
- 红包模板
- 返奖分档
- 交互策略
- 风控配置
- 数据看板
- 红包查询
- 返奖记录
- 风控审核
- 红包黑名单
- 飞书预警
- 操作日志

### 4.1 基础配置页

配置项包括：玩法开关、红包入口财富等级门槛、锁房是否允许发红包、红包倒计时、飘屏跳房开关、全服飘屏金额门槛、发送方自领是否允许。

### 4.2 红包模板页

金币红包模板：总金额档位、份数档位、领取条件、发放类型、是否展示返奖说明。  
礼物红包模板：礼物池、礼物单价、数量选项、可领取份数选项、领取条件、礼物墙展示开关。

### 4.3 返奖分档页

核心表格：

| 档位 | 金额区间 | 概率 | 比例 | 状态 |
| --- | --- | --- | --- | --- |
| A 档 | 20k - 49,999 | 3% | 5% | 开 |
| B 档 | >= 50k | 5% | 10% | 开 |

附加配置：是否仅领完后开奖、有效领取人数最低值、发送人自领是否计入、每日预算上限、预算触顶处理。

### 4.4 交互策略页

重点配置：

- 是否合并发送方结果弹窗与返奖弹窗。
- 合并策略：合并成一张卡，或结果后 1s 再展示返奖。
- 多弹窗队列上限。
- 超限降级方式：toast、红点、消息中心。
- 红包列表排序优先级。

### 4.5 风控配置页

重点配置：

- 同设备、多账号、同 IP 检测开关。
- 关联设备/IP 同步限制开关。
- 高频大额阈值。
- B 档聚集观察和强风险阈值。
- 自然互动热度 : 红包领取热度阈值，默认 <= 3:7。
- 领取随机延迟窗口，默认 1 - 3 秒。
- `< 500ms` 可疑领取标记开关。
- 连续 N 次 `< 1s` 触发验证码或冷却阈值。
- 生产环境模拟器限制开关。
- 飞书机器人 webhook 和告警群配置。

## 5. 已拆出的红包后台页面记忆点

### 5.1 红包数据查询页

文件：`hongbao-admin-pages/06-packets.html`，字段文档：`06-packets-field-doc.md`。

页面定位：

- 按红包 ID 维度查看发送、领取、返奖、风控、预算扣减的完整状态。
- 承接三个操作入口：详情、领取记录、风控。
- 红包 ID 是唯一开奖主键，也是排查、返奖幂等、补发队列的核心关联字段。

关键口径：

- 达标红包：金额或礼物总价值命中 A/B 档。
- 领完：有效期内全部份额被领取，且未被风控拦截时才进入返奖开奖。
- 预算影响：已中奖为正向扣减；未触发或风控拦截为 0；待复核可展示冻结。

列表核心字段：

- 红包 ID
- 发送人 / 房间
- 红包类型
- 金额 / 礼物
- 领取配置
- 领取进度
- 返奖资格
- 返奖状态
- 风控状态
- 预算影响
- 发送时间
- 操作：详情、领取记录、风控

验收重点：

- 已中奖红包详情展示完整返奖详情。
- 未达到返奖条件时，详情上半部分展示领取情况，下半部分展示“未达到返奖条件”。
- 未中奖、风控拦截、待人工复核时，返奖详情只展示“未中奖”。
- 点击风控跳转风控明细页并打开手动加黑弹窗，带入红包 ID、发送人 UID、房间 ID。

### 5.2 红包领取记录页

文件：`hongbao-admin-pages/09-claim-records.html`，字段文档：`09-claim-records-field-doc.md`。

页面定位：

- 按领取单号维度查看用户领取红包的完整流水。
- 排查领取是否成功、领取内容、领取条件、发送人自领、公屏消息、结果弹窗、钱包变账、风险标记。

关键口径：

- 领取单号是单笔领取流水主键。
- 发送人自领保留在明细中，但不计入返奖开奖所需的有效领取人数。
- 礼物红包领取内容展示礼物名称、数量和等价钻石。
- 响应耗时不在当前列表直接展示，但用于顶部统计、风险标记和风控计算。
- `claim_not_before` 必须由服务端校验，客户端倒计时只做展示。

列表核心字段：

- 领取单号
- 红包信息
- 领取人
- 领取内容
- 领取条件
- 消息 / 变账
- 风险标记
- 领取时间
- 操作或风控状态

交互：

- 普通记录点击“风控”跳转风控明细页并打开手动加黑弹窗。
- 已风控记录操作列展示危险级、限制级或观察名单；点击可调整等级。
- 风控跳转带入红包 ID、领取单号、领取人 UID、房间 ID、来源。

### 5.3 红包风控明细页

列表核心字段：

- 风险单号。
- 用户 / 房间。
- 关联红包。
- 命中规则。
- 证据摘要。
- 风控状态。

列表核心字段：

- 红包 ID / 开奖主键。
- 发送人 / 房间。
- 红包金额与档位。
- 开奖条件：仅展示已领完。
- 返奖金额。
- 发放状态。
- 风控状态。
- 结算时间。
- 操作。

返奖结算链路：

1. 红包领完。
2. 档位确认。
3. 金额计算。

验收重点：

- 顶部今日开奖红包数量应与列表中已领完返奖记录口径一致。
- 普通记录展示“风控”；已风控记录展示危险级、限制级或观察名单。
- 点击风控跳转风控明细页并打开手动加黑弹窗。
- 调整风控等级后，同步更新当前行风控状态和操作列。

## 7. 龙蛋玩法后台记忆点

### 7.1 核心定位

龙蛋玩法后台按分区独立配置和生效。后台核心解决两件事：

- 配置每个分区 / 房间可用的龙蛋奖励规则与 BUFF 规则。
- 查看指定房间在指定日期内的龙蛋达成情况、贡献榜单与 BUFF 期间礼物消耗数据。

默认口径：

- 每个房间每天按 GMT+3 00:00 重置龙蛋进度。
- 后端为最终口径，前端只做查询与运营配置。

### 7.2 信息架构

一级菜单：`龙蛋玩法`。  
二级菜单：奖励配置、BUFF 配置、房间数据看板、获奖记录、操作日志。

本次核心页面：

- 奖励配置模块。
- 数据看板模块。

### 7.3 奖励配置模块

重点能力：

- 按分区独立配置。
- 支持配置 Level1 - Level5。
- 支持奖励 ID 1 - 6 等不同奖励组合。
- 支持 Top10 贡献用户奖励与房主奖励。
- 支持 BUFF 倍率、时长、生效范围配置。
- 支持版本管理、发布、复制、停用。

校验规则：

- 停用配置不可编辑，只能复制生成新版本。
- 生效中配置修改后不直接覆盖，需生成新版本并设置生效时间。
- 同一分区生效时间冲突禁止提交。
- Level 门槛必须递增。
- 排名奖励不能为空。
- 奖励 ID 必须存在于奖励中心。
- BUFF 倍率不得小于 1，时长必须大于 0。

### 7.4 数据看板模块

重点能力：

- 查看房间基础信息。
- 查看房间当日最高等级龙蛋。
- 查看各等级完成状态。
- 点击不同等级查看贡献详情榜单。
- 查看发奖状态与获奖结果。
- 查看 BUFF 加成期间礼物消耗汇总。
- 查看单次 BUFF 明细、趋势和用户消耗数据。

关键口径：

- 统计日：GMT+3 00:00 - 23:59:59。
- 龙蛋最高等级：房间在统计日内已完成的最高 Level。
- 贡献值：统计周期内用户为当前等级累计的有效积分。
- 榜单排名：贡献值倒序；贡献值相同按达成时间升序。
- 用户中途离房仍保留榜单并可获奖。
- 参与人数不足 10 时按实际人数发奖。
- BUFF 期间礼物消耗：BUFF 生效窗口内产生的礼物订单总消耗。

## 8. CP / Soul Pair 玩法记忆点

### 8.1 核心定位

CP 玩法面向 MENA 语聊房，对外建议使用 `Soul Pair` 或 `Partner Bond`，`CP` 作为内部简称。一个用户同一时间仅可拥有 1 个 CP 关系。后端为准，前端只负责展示和交互承接。

后台目标：

- 运营可独立配置 CP 关系门槛和成长节奏。
- 商业化团队可独立配置专属礼物与礼包场景。
- 数据团队可追踪结 CP、成长、礼物消耗和解除风险。

适用范围：

- 分区。
- 国家 / 语言。
- 房间类型：公共语聊房、主播房、派对房、游戏房。
- 用户层级：全量、白名单、实验组。

### 8.2 后台信息架构

一级菜单：`CP玩法`。  
二级菜单：

- 基础配置。
- 关系规则。
- 亲密值规则。
- CP 等级。
- 专属礼物。
- 任务与纪念日。
- 房内展示。
- 风控审核。
- 数据看板。
- 关系记录。
- 操作日志。

### 8.3 关系与成长

关系建立规则记忆点：

- 单用户最大关系数默认 1。
- 账号注册时长门槛默认 >= 3 天。
- 可配置互关要求、同房互动要求、誓约礼物、确认有效期、房内/私聊发起入口。
- 发起后确认有效期默认 120 秒。

解除规则记忆点：

- 建立后保护期默认 24 小时。
- 可配置单方解除、二次确认、解除后冷却期、是否清空亲密值、是否保留纪念记录。
- 解除后冷却期默认 7 天。

亲密值来源：

- 同房在线。
- 双上麦。
- 私聊消息。
- 私聊语音。
- 普通礼物。
- CP 专属礼物。
- 日常任务。
- 纪念日任务。

默认等级建议：

1. 初识。
2. 默契。
3. 陪伴。
4. 守护。
5. 星月契约。

### 8.4 专属礼物与本地化

推荐礼物方向：

- 誓约戒 Moon Ring：发起结 CP。
- 沙漠玫瑰 Desert Rose：日常礼物。
- 双月灯 Twin Lantern：协同礼物。
- 绿洲王座 Oasis Throne：房间氛围展示。
- 星夜马车 Star Carriage：升级庆祝。
- 永恒契约 Eternal Bond：5 级 CP 专属。

MENA 本地化：

- 默认文案避免过强婚恋承诺表达，更偏陪伴、默契、守护、灵魂搭子。
- 视觉主题优先星月、绿洲、灯笼、Majlis、沙漠玫瑰。
- 文案和主题需支持阿语 RTL。
- 礼物命名避免低俗或过度暧昧。
- 保守分区可关闭房内公开秀关系动画，仅保留私密展示。

### 8.5 前台原型记忆点

文件：`cp-play-frontend-preview.html`。

前台流程覆盖：

- 房间内发现关系入口。
- 发起结 CP。
- CP 主页。
- 专属礼物商城。
- 升级庆祝与房效。

核心设计意图：

- 当用户已经同房、同麦、互相送礼时，结 CP 的转化阻力最低。
- 房间入口、发起页、CP 主页、礼物页、庆祝页已经足够验证收入和留存。
- 结 CP、升级、纪念日、房主房冲氛围、CP PK 都是核心消费节点。

## 9. 给后续 agent 的工作原则

- 继续做红包相关页面时，优先遵循 `hongbao-rebate-prd-v2.md` 和 `hongbao-admin-pages/*-field-doc.md` 的口径。
- 红包 ID 必须贯穿红包查询、领取记录、返奖记录、风控记录，是排查链路和开奖幂等的核心。
- 不要把“领取人获得内容”和“发送人返奖币种”混淆：礼物红包领取人拿礼物/钻石，发送方返奖拿金币。
- 不要把发送人自领计入有效领取人数；但领取明细必须保留发送人自领记录。
- 不要让未领完红包进入返奖抽奖链路。
- 命中限制级用户仍可正常发红包和领红包，但其发出的达标红包不参与概率返奖。
- 危险级用户应隐藏红包入口，并在服务端拒绝发红包和领红包接口。
- 风控和预算都应该发生在服务端，前端只展示状态和操作入口。
- 涉及弹窗时，优先合并同一红包的结果态，并遵守队列上限与优先级。
- 涉及数据看板时，必须区分 A 档和 B 档表现，并同时看补贴率、风控命中和交互质量。
- 做龙蛋页面时，始终按分区独立、GMT+3 统计日、后端结算为准。
- 做 CP 页面时，始终考虑 MENA 本地化、阿语 RTL、关系表达克制、灰度投放和保守分区差异。
