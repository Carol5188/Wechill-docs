异常标记
5 类 收入为 0、异常退出、未还预支、健康度异常、余额异常等统一打标签。

字段口径说明

  • 钻石流水表示周期内累计收入，不等于余额。
  • 起始/最终钻石余额属于时点值，用于描述周期内库存变化。
  • 工资余额是周期总工资扣除预支/借支后的实时余额。

筛选条件优化

大区筛选替代原有国家筛选，同时补充状态、健康度和是否存在预支/借支等条件，保证运营可以快速圈定目标人群。

异常提示策略

当出现“起始余额大于最终余额但无结算/提现记录”“有收入但总工资为 0”“存在未归还预支仍可退出”等情况时，列表需标红并支持跳转异常页。

  • 今日提现金额
  • 提现成功率
  • Coins 兑换金额

预支与借支

用于追踪未归还欠款，拦截“未还款仍退出 / 提现”的异常行为。

  • 未归还金额
  • 当前周期应发工资
  • 连续多周期未还清

提现记录

按时间倒序展示，支持筛选状态与渠道，并保留失败原因与收款账户脱敏信息。

  • 处理中 / 成功 / 失败 / 驳回

账务预警

面向财务的异常规则：提现失败、预支超额、借支接近上限、可提现余额异常等。

  • 异常标签汇总
  • 支持跳转对应记录

原型说明

这份 HTML 是面向“后台评审”的静态原型，不接真实接口。当前已经把 PRD 中最重要的结构、字段和视觉层级做进页面里。如果下一步你要，我可以继续把它拆成更像真实后台的多页面版本，或者直接接着补成 Vue / React 的管理后台页面骨架。
