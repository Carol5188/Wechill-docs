## 2. 页面定位

公会薪资页面用于从“公会维度”查看经营与薪资情况，重点关注：

1. 公会成员规模和高流水主播情况。
2. 公会钻石余额与钻石流水变化。
3. 公会工资结构，包括主播工资、公会长工资、公会长作为主播的收入影响。
4. 公会预支、借支和兑钻情况。

该页面更适合作为运营管理者、财务或公会运营的一级查询页面。

## 3. 页面规则总述

### 3.1 时间规则

- 时间筛选按“天”展示，不精确到时分秒。
- 默认展示格式为：`YYYY-MM-DD ~ YYYY-MM-DD`
- 页面实际查询时，默认按所选日期的 `00:00` 起算。

#### 3.1.1 字段统计口径规则

- `流水>30w主播数`、`起始钻石余额`、`最终钻石余额`、`钻石变化`、`兑换钻石` 这 5 个字段按筛选日期的起止点或日期范围统计。
- `流水>30w主播数` 统计筛选日期范围内钻石流水大于 `300,000` 钻石的主播人数。
- `起始钻石余额` 取筛选开始日期对应查询起点的公会成员钻石余额汇总。
- `最终钻石余额` 取筛选结束日期对应查询终点的公会成员钻石余额汇总。
- `钻石变化 = 最终钻石余额 - 起始钻石余额`。
- `兑换钻石` 统计筛选日期起止范围内该公会成员发生的兑钻金额汇总。
- 除上述 5 个字段外，其他字段按结算周期统计；如果筛选时间跨多个周期，则展示命中周期的合计值。例如筛选 `3月1日-3月31日`，其他字段展示 `3月1日-3月15日` 与 `3月16日-3月31日` 两个周期的总和。

### 3.2 公会维度聚合规则

公会薪资页面中的大多数数值字段为“公会维度聚合值”，即该公会内符合当前筛选条件的成员数据汇总：

- 人数字段：统计人数
- 钻石字段：统计汇总值
- 工资字段：统计金额汇总值
- 占比字段：由聚合值进一步计算得出

### 3.3 政策结构说明

`政策结构` 用于标识该公会内成员结算政策的整体结构情况，典型取值：

- `双周为主`
- `自由为主`
- `混合`

该字段用于快速判断公会成员结算模式，不等于单个成员的具体政策。

### 3.4 列表视觉规则

- `流水>30w主播数`、`起始钻石余额`、`最终钻石余额`、`钻石变化`、`兑换钻石` 这 5 个字段均为按筛选时间生效的字段，列表内容区统一使用浅蓝底色展示。
- 该底色用于区分“筛选时间直接生效的字段”和“筛选时间所在结算周期生效的字段”，不改变字段计算口径。
- 如果整行存在关注或异常底色，这 5 个字段仍保留同一套浅蓝字段底色以便横向核对。

## 4. 顶部文本汇总区字段说明

公会薪资页面顶部已改成与主播页面一致的“文本汇总 + 复制按钮”样式。

### 4.1 汇总区字段

| 字段 | 说明 |
| --- | --- |
| 公会总数 | 当前筛选命中的结算周期内存在经营数据的公会数量；跨周期时按去重公会统计 |
| 活跃主播数 | 当前筛选命中的结算周期内有开播或有收入的主播数量；跨周期时按去重主播统计 |
| 当前周期总工资 | 当前筛选命中的结算周期内平台公会工资总额，口径 = 主播工资总额 + 公会长工资总额；跨周期时按周期累加 |
| 预支借支总额 | 当前筛选命中的结算周期内的欠款总额，= 预支总额 + 借支总额；跨周期时按周期累加 |
| 预支总额 | 当前筛选命中的结算周期内未结清预支金额汇总；跨周期时按周期累加 |
| 借支总额 | 当前筛选命中的结算周期内未结清借支金额汇总；跨周期时按周期累加 |
| 主播工资总额 | 当前筛选命中的结算周期内所有主播工资合计，不含公会长作为主播的额外拆分口径；口径包含主播工资 + TG 奖励；跨周期时按周期累加 |
| 公会长工资总额 | 当前页面说明采用完整口径理解：公会长工资 + 公会长 TG 奖励 + 公会长作为主播的薪资 + TG 奖励；跨周期时按周期累加 |

### 4.2 复制按钮

- 按钮名称：`复制`
- 功能：复制当前文本汇总区内容，便于运营同步到日报、周报、群消息或表格。

## 5. 筛选项字段说明

当前页面只保留以下 7 个筛选项：

| 字段 | 说明 |
| --- | --- |
| 结算周期 | 按天选择查询范围，默认按选中日期 `00:00` 起算 |
| 快捷时间 | 固定快捷条件：`本周期 / 上一周期 / 今天 / 本周 / 上周` |
| 公会 ID | 精确查找单个公会 |

| 政策结构 | `全部 / 双周为主 / 自由为主 / 混合` |

## 7. 关键字段计算说明

### 7.1 钻石变化

`钻石变化 = 最终钻石余额 - 起始钻石余额`

### 7.2 公会总工资

建议按当前页面口径理解为：

`公会总工资 = 主播工资 + TG 奖励 + 公会长工资 + 公会长TG奖励`

如后续产品明确“公会长作为主播的收入”需单独包含或单独排除，应由后端统一调整。

### 7.3 公会应发总工资

`公会应发总工资 = 公会总工资 - 总预支 - 总借支`

### 7.4 公会长总工资 / 公会总工资的百分比

当前页面口径建议为：

`公会长总工资占比 = 公会长总工资 / 公会总工资 * 100%`

其中 `公会长总工资` 按本次对话补充说明理解为：

`公会长工资 + 公会长 TG 奖励 + 公会长作为主播的薪资 + TG 奖励`

## 8. 金额与单位说明

| 字段类型 | 单位 |
| --- | --- |
| 钻石类字段 | 钻石 |
| 工资 / 预支 / 借支类字段 | USD |

- 工资类字段建议统一保留两位小数。

## 9. 页面使用说明

### 9.1 适合回答的问题

公会薪资页面适合快速回答以下问题：

1. 某个公会当前周期赚了多少？
2. 这个公会在筛选时间内有多少流水大于 30w 钻石的主播？
3. 公会内工资结构里，公会长收入占比高不高？
4. 公会钻石余额是增长还是下降？
5. 公会是否存在较大的预支或借支风险？

### 9.2 推荐下钻路径

1. 点击公会进入公会详情，查看经营卡片与收入趋势。
2. 点击成员总数或流水>30w主播数，进入成员收入明细，再继续下钻到主播薪资页面。
3. 点击兑换钻石或钻石流水，核对余额变化和流水沉淀情况。
4. 点击总预支 / 总借支 / 公会应发总工资，跳转财务模块做对账和风险处理。

## 10. 与主播页面的关系

公会薪资页面与主播薪资页面的区别：

| 页面 | 重点 |
| --- | --- |
| 公会薪资页面 | 看公会整体经营、结构、收入占比、欠款风险 |
| 主播薪资页面 | 看单个账号的工资、钻石变化、兑钻、预支借支与操作动作 |

## 11. 待统一口径事项

1. `公会长总工资` 当前文档按“完整口径”解释，若后续产品只保留“公会长工资 + 公会长 TG 奖励”，需同步修改页面说明和后端口径。
2. `公会总工资` 与 `主播工资总额 / 公会长工资总额` 的拆分口径必须与结算引擎保持一致。
3. `兑换钻石` 仅统计兑钻金额，不混入其他支出口径。
