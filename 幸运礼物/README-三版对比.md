## 🎯 三版核心差异

| 维度 | A 基础版 | B 房间池版 | **C 共享池版** |
|---|---|---|---|
| 奖池维度 | ❌ 无 | 每房间独立 | **活动 × 国家共享** |
| 中奖资金来源 | 平台直接出 | 房间奖池+保底基金 | **共享池+有限补贴** |
| 平台成本控制 | RTP 单参锁定 | 双参+保底基金 | **结构性可控（池=用户钱）** |
| 概率模型 | 固定 | 固定+触发条件 | **动态（4 系数）** |
| 奖励形态 | 金币 | 金币 | **以虚拟奖励为主**（券/框/车） |
| 房间公平性 | ✅ 高 | ❌ 大房间远好于小房间 | ✅ 全国共享无差距 |
| 实现成本 | 3 周 MVP | 11 周分阶段 | 16 周（含多国家） |
| 合规风险（MENA） | 低 | **高**（房间奖池=赌博争议） | 中（虚拟化降险） |
| ARPU 拉动 | +15% | +25% | +20% |
| 推荐场景 | 长期常态 | 直播大主播 | **节日大型活动** |

## 💡 推荐选型

```
Phase 1（M1-M2）：A 基础版
  → 跑通基础链路，验证付费意愿

Phase 2（M3-M5）：A + C 组合
  → A 作为日常底盘
  → C 作为节日活动玩法（斋月/开斋节）

Phase 3（M6+）：常态化
  → A 日常 + C 节日 + 公会扩展
```

**为什么不推荐 B**：
1. 房间奖池导致大小房间体验严重不均
2. "房间奖池"概念在 MENA 法律定性上风险最高
3. 保底基金压力测试结果偏紧，财务不可控

## 🔑 三版共有的评审级要素

所有方案都包含：

- ✅ 数值脚本验证逻辑（保存配置时强制校验）
- ✅ RNG 安全细节（CSPRNG + HSM + 种子轮换 + 泄漏应急）
- ✅ RTP/概率调整流程（双签 + 灰度 + 审计）
- ✅ 客服 SOP（高频问题模板 + 工单升级 + 补发权限）
- ✅ 灰度回滚预案（阶段 + 触发条件 + 回滚方案）
- ✅ 异常人审流程（触发条件 + 24h SLA + 判定路径）
- ✅ 与红包/龙蛋的口径对齐（统计日/黑名单/审计）

## ⚠️ 评审会建议讨论的核心问题

1. **选型决策**：A / B / C / A+C 组合
2. **MENA 各国合规定性**（最大风险）：奖池+概率开奖的法律边界
3. **奖励虚拟化程度**：金币券 vs 装扮券 vs 完全不可消费
4. **保守地区差异化策略**：沙特/阿联酋是否完全关闭高档奖项
5. **苹果商店 loot box 合规**：是否需要中东版专项改造
6. **未成年人保护强度**：实名+二次验证
7. **大额中奖税务**：单笔 R5 等值是否触发当地税务申报
8. **客服补发自由裁量边界**：体验 vs 防薅羊毛
