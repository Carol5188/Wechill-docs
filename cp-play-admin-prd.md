# 语音房 CP 玩法后台配置 PRD（MENA 语聊房）

## 1. 文档说明

本文档用于补充语音房 `CP / Soul Pair / Partner Bond` 玩法的后台配置方案，供产品、设计、前后端、测试、运营使用。

目标：

1. 支持按分区、语言、房间类型配置 CP 玩法
2. 支持配置结 CP 条件、亲密值规则、等级体系、礼物、任务、纪念日、房内展示
3. 支持查看核心数据、异常行为、发奖与关系状态

默认定义：

- 对外文案建议使用 `Soul Pair` 或 `Partner Bond`
- `CP` 作为内部简称
- 一个用户同一时间仅可拥有 `1` 个 CP 关系
- 后端为准，前端只负责展示和交互承接

## 3. 配置目标与适用范围

### 3.1 配置目标

后台要解决三件事：

1. 让运营可以独立配置 CP 玩法的关系门槛和成长节奏
2. 让商业化团队可以独立配置专属礼物与礼包场景
3. 让数据团队可以清晰追踪结 CP、成长、礼物消耗和解除风险

### 3.2 适用范围

建议支持以下维度配置：

- 分区
- 国家/语言
- 房间类型
  - 公共语聊房
  - 主播房
  - 派对房
  - 游戏房
- 用户层级
  - 全量用户
  - 白名单用户
  - 指定实验组

## 5. 页面二：关系规则配置

### 5.1 页面目标

配置结 CP 的准入条件、发起流程、解除规则和冷却期。

### 5.2 关系建立规则

字段建议：

- 是否允许发起结 CP：开 / 关
- 关系名称：
  - Soul Pair
  - Partner Bond
  - 自定义
- 单用户最大关系数：默认 1
- 账号注册时长门槛：默认 `>=3天`
- 互关要求：是 / 否
- 同房互动要求：
  - 近7天同房总时长
  - 近7天同房双上麦时长
  - 近7天私聊语音时长
- 发起是否需要誓约礼物：是 / 否
- 誓约礼物ID
- 誓约礼物价格
- 发起后确认有效期：默认 120 秒
- 是否允许房内直接发起：是 / 否
- 是否允许私聊页发起：是 / 否

### 5.3 接受/拒绝规则

字段：

- 接收方是否可查看发起人资料摘要
- 接收方是否可查看关系权益预览
- 拒绝后是否允许当日再次发起
- 拒绝冷却时长
- 是否允许屏蔽某用户发起

### 5.4 解除规则

字段：

- 建立后保护期：默认 24 小时
- 是否允许单方解除：是 / 否
- 解除是否需二次确认：是 / 否
- 解除后冷却期：默认 7 天
- 冷却期内是否允许复绑原对象：是 / 否
- 解除是否清空亲密值：是 / 否
- 解除是否保留纪念记录：是 / 否

### 5.5 特殊关系审核

字段：

- 是否开启异常频繁换绑预警
- 近30天换绑次数阈值
- 命中后处理：
  - 仅告警
  - 禁止发起
  - 人工审核

## 6. 页面三：亲密值规则配置

### 6.1 页面目标

配置 CP 关系成长所依赖的亲密值获取和衰减规则。

### 6.2 亲密值来源配置

建议表格化配置：

| 行为类型 | 规则说明 | 亲密值 | 每日上限 | 是否计入连续天数 |
| --- | --- | --- | --- | --- |
| 同房在线 | 双方在同一房间 | +1/分钟 | 60 | 是 |
| 双上麦 | 双方同房且都在麦上 | +2/分钟 | 60 | 是 |
| 私聊消息 | 双方有效私聊消息 | +1/次 | 20 | 否 |
| 私聊语音 | 私聊语音时长 | +2/分钟 | 40 | 是 |
| 普通礼物 | 按礼物价值折算 | 价值 * 系数 | 200 | 否 |
| CP礼物 | 仅专属礼物 | 价值 * 高系数 | 500 | 是 |
| 日常任务 | 完成双人任务 | 固定值 | 任务维度 | 是 |
| 纪念日任务 | 纪念日完成行为 | 固定值 | 1次 | 否 |

### 6.3 亲密值系数配置

字段：

- 普通礼物折算系数
- CP 专属礼物折算系数
- 房主房加权系数
- 活动期间加成系数
- 新建关系前7天加成系数

### 6.4 亲密值衰减规则

字段：

- 是否开启衰减
- 连续未互动多少天开始衰减
- 每日衰减值
- 最低保留值
- 是否影响等级

### 6.5 连续陪伴天数规则

字段：

- 连续天数判定时间区间
- 计入连续天数的行为类型
- 中断后是否允许补签
- 补签道具开关
- 补签道具价格

## 19. MENA 本地化建议

- 默认文案避免过强“婚恋承诺”表达，更建议 `陪伴、默契、守护、灵魂搭子`
- 视觉主题优先使用 `星月、绿洲、灯笼、Majlis、沙漠玫瑰`
- 文案和主题需支持阿语 RTL 布局
- 礼物命名避免低俗或过度暧昧
- 在保守分区中可关闭房内公开秀关系动画，仅保留私密展示
