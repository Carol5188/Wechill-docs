v1.2 全链路风控规则补充
L0 正常注册、登录、充值、加入公会、收益结算均按常规链路放行。
L1 观察仅记录证据和风险分，不影响核心能力；观察期无复犯可自动降级。
L2 审核注册、加入公会、提现、收益进入审核，充值可按设备档位处理。
L3 危险限制主播、公会、提现能力，冻结风险链路收益，充值按设备限额拦截。
L4 封禁账号/设备/严重欺诈优先生效；白名单不可覆盖法务、合规和严重欺诈封禁。
注册校验 登录补判 主播/公会校验 设备充值累计 收益冻结 申诉解封 审计复盘
  • 单设备最多 3 个有效账号；同设备第 4 个注册拦截或进入人工审核，卸载重装、清缓存、广告 ID 变化均不重置统计。
  • 单设备只允许 1 个有效主播占用；主播退出公会、封禁、注销、未结算收益期间默认不立即释放名额。
  • 单 IP 24 小时最多 5 个注册账号；公共网络只作为辅助证据，VPN / 机房 IP 降低阈值或强验证。
  • 充值封禁以 risk_device_hash 为主口径，Apple、Google、三方支付按 USD 折算后合并累计。
  • 高危封禁、设备充值解封、收益解冻、批量删除、白名单变更必须记录审批、证据链和操作审计。

设备生命周期与绕过处理

覆盖卸载重装、清缓存、广告 ID 重置、游客转正、注销重注册、二手设备、维修换机、模拟器和 Root / 越狱设备等绕过场景。

  • risk_device_hash 稳定命中时，不因客户端字段变化重置账号数与充值累计。
  • 二手设备、维修换机只允许通过申诉和审批释放设备占用。

主播设备占用与释放

主播身份以“加入公会并具备主播能力”为占用起点，退出公会、封禁、注销、未结算收益、提现或申诉中均进入保留或冷却状态。

  • 默认冷却期 7 天；确认误判、二手设备或永久清退且无财务纠纷时可人工释放。
  • 同一主播 24h 登录多设备会标记账号共享风险。

公共 IP / VPN / 代理网络

IP 风险不单独作为永久封禁依据，公共 Wi-Fi、移动运营商 NAT、公会办公网络以强验证、观察和白名单为主，仍需联动设备证据。

  • IP 白名单只豁免 IP 限制，不豁免设备、账号、公会、收益、充值风控。
  • 机房 IP、VPN / Proxy 触发注册拦截、短信限制或提现审核。

申诉与证据链

后台证据链

  • 设备证据：同设备账号、主播、设备指纹变更。
  • IP 证据：同 IP 注册/登录、VPN、机房 IP。
  • 收益证据：工资、提成、互刷链路、提现记录。
  • 充值证据：设备累计、渠道拆分、退款/拒付。

用户申诉材料

  • 设备充值封禁：支付订单截图、支付凭证、充值账号说明。
  • 设备封禁：设备截图、登录账号说明、二手/维修证明。
  • 收益冻结：收益来源、公会证明、房间/活动说明。
  • 处理时效：普通 1-3 个工作日，收益/公会 3-7 个工作日。

规则优先级

  • 法务/合规强制封禁 > 严重欺诈/拒付 > 账号封禁。
  • 账号封禁 > 设备封禁 > 设备充值封禁 > 主播限制。
  • 收益冻结 > 提现限制 > IP 限制 > 观察规则 > 白名单。

恢复边界

  • 设备充值解封只恢复充值能力，不解除账号封禁、提现限制、收益冻结。
  • 账号解封不恢复主播身份；主播恢复不代表收益解冻。
  • IP 解封不代表设备和账号风险清零。

降级兜底

  • 低风险浏览 fail-open；资金、主播身份、提现、结算 fail-close。
  • 充值风控超时不创建订单；收益冻结失败则结算暂停。
  • 审批通过但执行失败保持待执行，并支持重试。
