## 2. 页面定位

主播薪资页面用于查询单个主播或公会长账号在指定时间范围内的薪资、钻石、预支借支、可提现余额等数据。

该页面的核心目标：

1. 让运营快速查看某个账号的收入构成。
2. 让财务核对工资、TG 奖励、预支、借支、余额、可提现金额。
3. 让风控快速识别异常账号，并执行封号或加入兑钻黑名单等后台动作。

## 3. 页面规则总述

### 3.1 时间规则

- 时间筛选按“天”展示，不精确到时分秒。
- 默认展示格式为：`YYYY-MM-DD ~ YYYY-MM-DD`
- 页面实际查询时，默认按所选日期的 `00:00` 起算。

#### 3.1.1 字段统计口径规则

- `起始钻石余额`、`最终钻石余额`、`钻石变化`、`兑换钻石` 这 4 个字段按筛选日期的起止点统计。
- `起始钻石余额` 取筛选开始日期对应查询起点的钻石余额。
- `最终钻石余额` 取筛选结束日期对应查询终点的钻石余额。
- `钻石变化 = 最终钻石余额 - 起始钻石余额`。
- `兑换钻石` 统计筛选日期起止范围内发生的兑钻金额。
- 除上述 4 个字段外，其他字段按结算周期统计；如果筛选时间跨多个周期，则展示命中周期的合计值。例如筛选 `3月1日-3月31日`，其他字段展示 `3月1日-3月15日` 与 `3月16日-3月31日` 两个周期的总和。

### 3.2 结算政策规则

#### 双周政策

- 每月 1 日结算上月 16 日至自然月最后一天。
- 每月 16 日结算当月 1 日至 15 日。
- 主播工资按钻石余额和后台梯度结算。
- TG 奖励按钻石流水和后台梯度结算。
- 任务要求为每周期 `6 天 * 2 小时`。
- 页面中“双周政策”的 `有效天` 小于 `6 天` 时，视为未达标，该行使用浅红底色展示。

#### 自由政策

- 主播可随时结算。
- 主播工资公式：`钻石余额 / 16026 * 75%`，结果保留两位小数并向下取整。
- 自由政策的 `有效天` 不参与达标判断，页面统一展示为 `-`。
- 自由政策即使有效天为空，也不使用浅红未达标样式。

### 3.3 用户身份规则

- 页面支持两种身份：
  - `主播`
  - `公会长`
- 当 `用户身份 = 主播` 时：
  - `公会长工资` 展示 `-`
  - `公会长TG奖励` 展示 `-`
  - `已借支` 展示 `-`
- 当 `用户身份 = 公会长` 时：
  - `公会长工资` 展示真实金额
  - `公会长TG奖励` 展示真实金额
  - `已借支` 展示真实金额

### 3.4 兑换钻石字段规则

- `兑换钻石` 属于按筛选日期起止点统计的字段。
- `兑换钻石` 只展示筛选日期范围内发生的兑钻金额。
- 不再混入“主播结算”“提现”“预支”“借支”等其他支出口径。
- 如无兑钻，则展示 `0`。

### 3.5 列表视觉规则

- `起始钻石余额`、`最终钻石余额`、`钻石变化`、`兑换钻石` 这 4 个字段为按筛选日期起止点统计的钻石余额与兑钻核心字段，列表内容区统一使用浅蓝底色展示。
- 该底色仅用于帮助运营快速识别钻石相关字段，不改变字段计算口径。
- 如果整行存在未达标或异常底色，这 4 个字段仍保留同一套钻石字段底色以便横向核对。

## 4. 顶部文本汇总区字段说明

页面顶部为“公会数据总览”文本汇总区，带复制按钮。

### 4.1 汇总区字段

| 字段 | 说明 |
| --- | --- |
| 公会钻石总流水（总收钻） | 当前筛选命中的结算周期内，纳入公会结算口径的所有主播/公会长礼物收钻总和；跨周期时按周期累加 |
| 公会总工资 | 当前筛选命中的结算周期内，所有公会相关账号工资总额；跨周期时按周期累加 |
| 公会总欠款 | 当前筛选命中的结算周期内未结清欠款总额，= 借支总额 + 预支总额；跨周期时按周期累加 |
| 借支总额 | 当前筛选命中的结算周期内未结清借支金额汇总；跨周期时按周期累加 |
| 预支总额 | 当前筛选命中的结算周期内未结清预支金额汇总；跨周期时按周期累加 |
| 公会总余额 | 当前筛选命中的结算周期内公会剩余可结算金额，公式为 `公会总工资 - 公会总欠款`；跨周期时按累计公会总工资 - 累计公会总欠款计算 |
| 公会总兑钻 | 按筛选日期起止范围统计的全部账号兑钻金额汇总 |
| 公会总工资 / 总流水（充值）的百分比 | 用于评估工资成本相对流水规模的占比；跨周期时按累计工资 / 累计流水计算 |

### 4.2 复制按钮

- 按钮名称：`复制`
- 功能：复制当前文本汇总区的全部内容，便于运营直接发送到 IM、表格或日报中。

## 8. 计算与判断规则

### 8.1 钻石变化

`钻石变化 = 最终钻石余额 - 起始钻石余额`

### 8.2 工资余额

`工资余额 = 总工资 - 已预支 - 已借支`

### 8.3 双周政策达标判断

- `有效天 >= 6`：达标，不做浅红异常展示。
- `有效天 < 6`：未达标，整行浅红展示。

### 8.4 自由政策有效天

- 页面固定展示 `-`
- 不参与浅红未达标规则

## 9. 操作按钮说明

当前页面操作统一为 3 个按钮：

| 按钮 | 说明 |
| --- | --- |

## 10. 操作弹窗说明

### 10.1 查看详情

- 弹窗名称：`主播薪资详情`
- 用途：查看单个主播或公会长账号在当前筛选条件下的收入构成、钻石变化和关键财务状态；其中钻石余额与兑钻按筛选日期起止点统计，其他字段按命中结算周期统计
- 建议展示内容：
  - 主播 ID / 身份
  - 当前周期总工资
  - 起始 / 最终钻石余额
  - 兑换钻石
  - 账变入口

### 10.2 封号

- 弹窗名称：`封号确认弹窗`
- 用途：对异常账号执行封号操作
- 建议展示内容：
  - 封禁类型
  - 封禁时长
  - 触发原因
  - 备注
  - 关联异常记录

### 10.3 添加兑钻黑名单

- 弹窗名称：`添加兑钻黑名单弹窗`
- 用途：限制用户后续兑钻行为
- 建议展示内容：
  - 限制范围
  - 生效状态
  - 拉黑原因
  - 备注

## 11. 异常识别规则

页面需重点关注以下情况：

1. 双周政策下 `有效天 < 6 天`。
2. 有收入但总工资为 0。
3. 存在未归还预支且发起退出申请。
4. 公会长身份出现已借支金额时，需同步核对公会长工资、公会长TG奖励与工资余额。
5. 起始余额大于最终余额但无合理账变记录时，需进一步核对。

## 12. 待统一口径事项

1. `当期钻石流水` 涉及自赠场景时，必须与结算引擎统计口径保持一致。
2. `已预支` 是展示当前周期已发生金额还是当前未归还金额，最终需以后端统一输出口径为准。
