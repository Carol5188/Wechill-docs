# 红包领取记录页面字段说明文档

版本日期：2026-04-18  
对应页面：`hongbao-admin-pages/09-claim-records.html`  
适用对象：产品、后台研发、服务端研发、测试、运营、客服、风控审核人员

## 1. 页面定位

红包领取记录页面用于按领取单号维度查看用户领取红包的完整流水，帮助运营、客服和风控人员排查以下问题：

- 用户是否成功领取红包。
- 领取人获得的是金币、礼物还是等价钻石。
- 领取人是否满足在房、在麦等领取条件。
- 发送人自领是否计入有效领取人数。
- 公屏消息、结果弹窗、钱包变账是否产生。
- 是否需要将领取用户送入风控处理。
- 已风控领取记录当前处于危险级、限制级还是观察名单。

本页面以领取单号为明细主键，以红包 ID、发送人 UID、领取人 UID、房间 ID、领取条件、消息变账、风险标记和风控状态作为主要排查维度。

## 2. 关键口径

| 口径 | 说明 |
| --- | --- |
| 领取单号 | 单笔领取流水的唯一标识。每一次用户尝试领取红包都应生成或关联一条领取记录。 |
| 有效领取 | 可参与红包领完判断和返奖开奖口径的领取记录。发送人自领、失败领取、不满足条件领取不计入有效领取人数；当前列表和选中详情不展示单条有效领取字段。 |
| 发送人自领 | 发送人领取自己发出的红包。记录保留在明细中，但不计入有效领取人数。 |
| 金币红包领取内容 | 领取人实际获得的金币数量。 |
| 礼物红包领取内容 | 领取人获得的礼物名称、数量和等价钻石；发送方返奖仍以金币结算。 |
| 响应耗时 | 从红包发出或服务端允许领取后，到领取请求完成的耗时。当前不作为筛选项或列表字段展示，仅用于顶部统计、风险标记、后端日志和风控计算。 |
| 随机延迟窗口 | 红包发出后，服务端设置 1 - 3 秒随机延迟，`claim_not_before` 之前不允许领取。 |
| 风控入口 | 普通领取记录可点击“风控”跳转红包风控明细页，并打开手动加黑弹窗。 |
| 风控状态 | 已被风控的领取记录不展示风控按钮，直接展示危险级、限制级、观察名单等状态标签。 |

## 3. 顶部数据总览

| 字段 | 含义 | 计算/来源 | 展示说明 |
| --- | --- | --- | --- |
| 今日领取次数 | 当日红包领取请求或领取流水总次数。 | 按领取单号统计。包含成功、失败、不满足条件等领取尝试。 | 当前展示 `6,842`，副文案拆分金币红包和礼物红包领取次数。 |
| 有效领取人数 | 当日可计入返奖开奖口径的去重领取人数。 | 成功领取且符合有效领取口径的用户去重统计。发送人自领不计入。 | 当前展示 `5,736`。 |
| 平均领取响应 | 当日领取响应耗时平均值。 | 领取耗时总和 / 领取次数，可按成功领取或全部请求分别统计。 | 当前展示 `2.8s`，副文案提示 1 - 3 秒随机延迟窗口。 |
| 异常响应样本 | 当日被标记为异常响应的领取样本数。 | 领取响应小于 500ms，或连续多次小于 1s 的记录。 | 当前展示 `26`，用于防脚本抢红包排查。 |

## 4. 筛选区字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| 领取时间 | 时间范围输入 | 按领取发生时间筛选。默认展示当天 `00:00 - 23:59`。 |
| 红包 ID | 输入框 | 精确查询某个红包下的领取流水。 |
| 发送人 UID | 输入框 | 按红包发送人筛选，可排查发送人自领、固定团伙互抢等行为。 |
| 领取人 UID | 输入框 | 按领取人筛选，可排查某个用户的抢红包行为。 |
| 红包类型 | 下拉 | 可选全部、金币红包、礼物红包。 |
| 风控状态 | 下拉 | 可选全部、危险级：禁发禁领、限制级：不参与返奖、观察名单、正常。 |
| 重置 | 按钮 | 清空筛选条件并恢复默认查询口径。 |
| 查询 | 按钮 | 按当前筛选条件刷新列表和顶部统计数据。 |

## 5. 领取记录列表字段

| 字段 | 含义 | 展示样式/示例 | 研发备注 |
| --- | --- | --- | --- |
| 领取单号 | 单笔领取流水号。 | `CLAIM-771901`，副文本可展示 `第 1 份`、`最后一份`、`失败样本`、`异常样本`。 | 建议作为领取明细主键，用于客服和风控追溯。 |
| 红包信息 | 领取记录关联的红包 ID、红包类型、返奖档位和房间 ID。 | 主文本 `HB202604160012`，副文本 `金币红包 / A档 / 房间 880321`。 | 应可拆分为红包 ID、红包类型、返奖档位、房间 ID。 |
| 领取人 | 领取用户 UID 和用户昵称/身份备注。 | 主文本 `UID 112345`，副文本 `Noura`、`发送人自领`。 | 发送人自领需保留明细，用于返奖口径和客服追溯。 |
| 领取内容 | 用户本次领取得到的内容。 | 金币红包：`2,456 金币`；礼物红包：`星光 × 1 / 999 钻石`；失败记录为 `-`。 | 礼物红包需要展示礼物名称、数量、等价钻石。 |
| 领取条件 | 当前红包要求的领取资格。 | `在房用户`、`在麦用户`。 | 服务端应按房间在线状态、麦位状态等条件校验。 |
| 消息 / 变账 | 本次领取触发的前端消息和钱包变账摘要。 | `公屏已发 / 钱包：红包获得 +2,456`、`Toast 已发 / 该红包只能由麦上用户领取`。 | 用于定位公屏消息、Toast、结果弹窗、钱包流水是否一致。 |
| 风险标记 | 本次领取关联的风险标签。 | `正常`、`高风险房间`、`< 500ms 可疑`。 | 可作为跳转风控或黑名单处理的依据。 |
| 领取时间 | 本次领取发生时间。 | `2026-04-16 10:08:45`。 | 建议后端返回秒级时间。 |
| 操作 | 当前领取记录可执行动作或风控状态。 | 普通记录展示 `风控`；已风控记录展示 `危险级：禁发禁领`、`限制级：不参与返奖`、`观察名单`。 | 点击风控跳转风控明细页并打开手动加黑弹窗；点击已风控状态可调整等级。 |

## 6. 当前弹窗与操作交互

| 操作/状态 | 交互 | 结果 |
| --- | --- | --- |

| 危险级：禁发禁领 | 点击后打开调整风控等级弹窗。 | 可切换为危险级、限制级或观察名单。 |
| 限制级：不参与返奖 | 点击后打开调整风控等级弹窗。 | 可切换为危险级、限制级或观察名单。 |
| 观察名单 | 点击后打开调整风控等级弹窗。 | 可切换为危险级、限制级或观察名单。 |
| 导出 | 导出当前筛选结果。 | 建议导出列表全部字段，并附带筛选条件、导出人、导出时间。 |

### 6.1 风控弹窗带入字段

领取记录页点击“风控”后，会跳转到红包风控明细页，并自动打开“手动加黑名单”弹窗。

| 带入字段 | 来源 |
| --- | --- |
| 红包 ID | 列表中的红包信息字段。 |
| 领取单号 | 列表中的领取单号字段。 |
| 领取人 UID | 列表中的领取人字段。 |
| 房间 ID | 列表中的红包信息字段。 |
| 来源 | 固定带入人工复核。 |

### 6.2 调整风控等级弹窗

已进入风控状态的领取记录，在操作列点击当前状态后打开“调整风控等级”弹窗。

| 字段/选项 | 说明 |
| --- | --- |
| 弹窗标题 | 固定展示“调整风控等级”。 |
| 弹窗副标题 | 展示当前领取单号和领取人 UID，方便确认调整对象。 |
| 危险级：禁发禁领 | 用户不可发红包、不可领取红包。 |
| 限制级：不参与返奖 | 用户可发可领，但发送的达标红包不参与高额红包概率返奖。 |
| 观察名单 | 只做监控和预警，不直接拦截。 |
| 取消 | 关闭弹窗，不修改当前风控状态。 |
| 选择任一等级 | 更新当前领取记录操作列中的风控状态展示。 |

## 11. 风控与日志事件建议

| 事件 | 触发场景 |
| --- | --- |
| `redpacket_claim_success` | 用户领取红包成功。 |
| `redpacket_claim_failed` | 用户领取失败、红包已过期或来晚一步。 |
| `redpacket_claim_condition_denied` | 用户不满足在房、在麦等领取条件。 |
| `redpacket_claim_sender_self` | 发送人领取自己发出的红包。 |
| `redpacket_claim_too_fast` | 领取响应小于 500ms。 |
| `redpacket_claim_captcha_trigger` | 连续快速领取触发验证码。 |
| `redpacket_claim_cooldown_trigger` | 连续快速领取触发临时冷却。 |
| `redpacket_claim_message_sent` | 公屏消息或 Toast 发送成功。 |
| `redpacket_claim_wallet_changed` | 领取导致钱包变账成功。 |
| `redpacket_claim_risk_review_opened` | 运营从领取记录发起风控处理。 |

## 12. 测试验收点

- 顶部今日领取次数应与列表筛选口径一致。
- 筛选区不展示领取状态、响应耗时筛选项，应展示风控状态筛选项。
- 列表不展示领取结果、响应耗时、有效领取字段。
- 发送人自领记录应展示在列表中，并在选中详情中保留返奖口径说明。
- 礼物红包领取内容应展示礼物名称、数量和等价钻石。
- 不满足领取条件的记录领取内容应为空或 `-`，消息 / 变账应展示 Toast 或失败原因。
- 领取响应 `< 500ms` 的记录应标记为可疑。
- 已进入风控的记录，操作列应展示危险级、限制级或观察名单状态，而不是风控按钮。
- 点击已风控状态，应打开调整风控等级弹窗，并支持切换危险级：禁发禁领、限制级：不参与返奖和观察名单。
- 普通记录点击风控，应跳转红包风控明细页并打开手动加黑弹窗。
- 风控跳转应带入红包 ID、领取单号、领取人 UID、房间 ID 和来源。
- 导出结果应包含列表展示字段，并保留当前筛选条件。
- 领取链路中 `claim_not_before` 必须由服务端校验，客户端倒计时只做展示。
