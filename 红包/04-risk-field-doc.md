# 红包风控明细页面字段说明文档

版本日期：2026-04-18  
对应页面：`hongbao-admin-pages/04-risk.html`  
适用对象：产品、后台研发、服务端研发、测试、运营、风控审核人员

## 1. 页面定位

红包风控明细页面用于查看红包玩法中的风险命中记录，并支持运营对风险用户进行人工处理。页面覆盖以下核心场景：

- 自动风控命中：同设备 / 同 IP 互抢、高频大额红包、B 档房间聚集、高风险房间、生产模拟器、毫秒级领取等。
- 飞书预警：针对高频大额红包、B 档房间聚集、高风险房间等异常行为发送预警。
- 手动加黑：运营巡房、用户投诉、人工复核后，将用户加入危险级、限制级或观察名单。
- 黑名单处理：对已命中记录执行解除、维持、调整等级等操作。

本页面以风险单号为风控审核主键，以用户 UID、房间 ID、红包 ID、设备/IP 为主要关联维度。

## 2. 关键口径

| 口径 | 说明 |
| --- | --- |
| 风险单号 | 单条风控命中记录的唯一标识。自动命中记录使用 `RISK-xxxxx`，手动新增记录使用 `RISK-MANUAL-xxxxx`。 |
| 危险级 | 最高风险等级。用户不展示红包入口，禁发禁领。适用于模拟器、明确作弊、严重团伙套利等。 |
| 限制级 | 中风险等级。用户可发可领，但其发出的达标红包不参与概率返奖。适用于关联设备/IP、高风险房间发送人、高频大额用户等。 |
| 观察名单 | 低风险或待观察状态。只记录、监控和预警，不直接拦截红包入口、发放、领取或返奖。 |
| 手动加黑 | 运营通过弹窗人工提交的黑名单记录。提交后按照操作时间插入风控命中明细列表。 |
| 关联设备/IP 同步限制 | 命中同设备或高风险 IP 后，该设备/IP 下关联账号同步限制返奖资格。 |
| 飞书消息提醒 | 当前风险记录是否已通过飞书机器人通知预警群。当前页面仅展示已发送或未发送。 |
| 审核状态 | 风控记录当前处理状态，当前页面统一为待审核和已风控两类。 |

## 6. 手动加黑名单弹窗字段

手动加黑入口位于风控命中明细右上角。点击“＋ 手动加黑”后打开弹窗。提交成功后，记录会按照操作时间插入风控命中明细列表，并同步更新今日新增黑名单数量。

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| 用户 UID | 是 | 被加入黑名单或观察名单的用户 ID。为空时展示错误提示“请填写用户 UID”。 |
| 黑名单等级 | 是 | 可选危险级、限制级、观察名单。默认限制级。 |
| 来源 | 是 | 可选运营巡房、用户投诉、人工复核、关联设备/IP。默认运营巡房。 |
| 关联房间 ID | 否 | 风险发生或运营巡房所在房间。若从红包数据查询页跳转，可自动带入。 |
| 关联红包 ID | 否 | 与本次加黑有关的红包 ID。若从红包数据查询页或领取记录页发起，可自动带入。 |
| 有效期 | 是 | 可选 7 天、30 天、永久。默认 7 天。 |
| 加黑原因 | 否 | 运营填写的风险原因。未填写时系统可根据来源、等级、有效期生成默认原因。 |
| 取消 | - | 关闭弹窗，不提交数据。 |
| 确认加黑 | - | 校验用户 UID 后提交，并按操作时间插入一条手动加黑记录。 |

### 6.1 手动新增后列表展示

| 列表字段 | 展示口径 |
| --- | --- |
| 风险单号 | 生成 `RISK-MANUAL-xxxxx`，副文本展示 `手动添加`。 |
| 用户 / 房间 | 展示 `UID 用户ID`、等级标签、来源标签、有效期标签和房间 ID。 |
| 关联红包 | 有红包 ID 时展示红包 ID，副文本为人工关联红包；无红包 ID 时展示手动加黑，副文本为无关联红包。 |
| 命中规则 | 根据等级展示手动加黑标签。 |
| 证据摘要 | 展示运营填写的加黑原因；未填时展示默认原因。 |
| 风控状态 | 根据运营选择展示危险级：禁发禁领、限制级：不参与返奖或观察名单。 |
| 飞书消息提醒 | 展示未发送。 |
| 审核状态 | 展示已风控。 |
| 时间 | 展示提交操作时间，并作为手动加黑记录插入列表的排序依据。 |
| 操作 | 仍可执行解除、维持、调整等级。 |

## 7. 风险操作弹窗字段

列表操作统一使用风险操作确认弹窗。弹窗标题、副标题、正文和按钮会根据操作类型变化。

### 7.1 解除

| 字段/按钮 | 说明 |
| --- | --- |
| 弹窗标题 | `确认解除黑名单？` |
| 弹窗副标题 | 风险单号 + 用户信息。 |
| 正文说明 | 确认后用户从黑名单移除，恢复红包入口、发放、领取及返奖资格；取消不做任何修改。 |
| 取消 | 关闭弹窗，不修改记录。 |
| 确认解除 | 审核状态保持已风控，更新时间为当前时间，并写入操作详情；风控状态字段保留原危险级、限制级或观察名单口径。 |

### 7.2 维持

| 字段/选项 | 说明 |
| --- | --- |
| 弹窗标题 | `选择维持时间` |
| 弹窗副标题 | 风险单号 + 用户信息。 |
| 正文说明 | 选择后确认维持当前风控状态，并更新该风险单有效期展示。 |
| 7 天 | 短期观察，适合首次命中或证据较轻。 |
| 30 天 | 中期限制，适合高频异常或团伙嫌疑。 |
| 永久 | 长期拦截，适合模拟器或明确作弊证据。 |
| 选择后结果 | 审核状态更新为已风控，时间更新为当前时间，并写入操作详情；风控状态字段不展示维持限制，只保留原危险级、限制级或观察名单口径。 |

### 7.3 调整等级

| 字段/选项 | 说明 |
| --- | --- |
| 弹窗标题 | `调整黑名单等级` |
| 弹窗副标题 | 风险单号 + 用户信息。 |
| 正文说明 | 选择等级后立即确认。 |
| 危险级 | 不展示入口，禁发禁领。 |
| 限制级 | 可发可领，但限制返奖资格。 |
| 观察名单 | 只告警和观察，不直接拦截。 |
| 选择后结果 | 更新列表行样式、风控状态、审核状态和时间，并写入操作详情。 |

## 12. 测试验收点

- 点击“＋ 手动加黑”应打开手动加黑名单弹窗。
- 用户 UID 为空点击确认加黑，应展示“请填写用户 UID”，且不新增列表记录。
- 选择危险级、限制级、观察名单提交后，应按操作时间插入对应样式的风控记录。
- 手动加黑成功后，今日新增黑名单数字应自增，列表总数应自增。
- 从红包数据查询页或领取记录页跳转风控时，应自动打开手动加黑弹窗，并带入红包 ID、用户 UID、房间 ID等参数。
- 点击解除，应弹出确认弹窗；确认后审核状态保持已风控，风控状态仍保留危险级、限制级或观察名单之一。
- 点击维持，应弹出 7 天、30 天、永久选项；选择后应更新有效期和操作详情，风控状态仍保留危险级、限制级或观察名单之一。
- 点击调整等级，应弹出危险级、限制级、观察名单选项；选择后应更新列表行样式和风控状态。
- 命中高频大额、B 档聚集强风险、高风险房间等预警类规则时，应有飞书消息提醒记录，且只展示已发送或未发送。
