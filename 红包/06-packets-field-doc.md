# 红包数据查询页面字段说明文档

版本日期：2026-04-18  
对应页面：`hongbao-admin-pages/06-packets.html`  
适用对象：产品、后台研发、服务端研发、测试、运营

## 1. 页面定位

红包数据查询页面用于按红包 ID 维度查看红包从发送、领取、返奖、风控到预算扣减的完整状态。页面同时承接三个操作入口：

- 详情：查看当前红包的详情概览和选中返奖详情。
- 领取记录：查看当前红包下的领取流水。
- 风控：跳转到红包风控明细页，并打开手动加黑弹窗。

红包 ID 是唯一开奖主键。服务端返奖结算、重试幂等、补发队列均应以红包 ID 作为核心关联字段。

## 2. 关键口径

| 口径 | 说明 |
| --- | --- |
| 有效红包 | 发送成功并进入领取链路的红包。不包含发送失败、创建失败、被业务取消的红包。 |
| 金币红包金额 | 用户实际发送的金币总额。领取人获得金币。 |
| 礼物红包金额 | 按礼物 ID 对应的金币价值乘以礼物数量折算。领取人获得等价钻石，发送方返奖获得金币。 |
| 达标红包 | 红包金额或礼物总价值命中返奖档位的红包。当前页面展示 A 档和 B 档。 |
| A 档 | 20,000 - 49,999 金币，示例返奖规则为 3% 概率返 5%。 |
| B 档 | 大于等于 50,000 金币，示例返奖规则为 5% 概率返 10%。 |
| 领完 | 红包全部份额在有效期内被领取。只有领完且未被风控拦截的达标红包才进入返奖开奖链路。 |
| 预算影响 | 当前红包对返奖预算的影响。已中奖为正向扣减，未触发或风控拦截为 0，待人工复核可展示冻结。 |

## 3. 顶部数据总览

| 字段 | 含义 | 计算/来源 | 展示说明 |
| --- | --- | --- | --- |
| 今日红包 GMV | 当日红包发送成功的总规模。 | 金币红包取发送金币金额；礼物红包取礼物总价值折算金币。 | 当前展示 `2.46M`，副文案拆分金币红包和礼物红包占比。 |
| 达标红包数 | 当日命中返奖门槛的红包数量。 | 统计 A 档、B 档红包数量。礼物红包按总价值命中分档。 | 当前展示 `1,259`，副文案展示 A 档和 B 档数量。 |
| 今日返奖金额 | 当日已发放或已补发成功的返奖金币总额。 | 与红包返奖记录页的已发放口径保持一致。 | 当前展示 `7,683`，副文案展示 A 档和 B 档返奖金额。 |
| 红包领完率 | 当日有效红包中已领完红包的占比。 | 已领完红包数 / 有效红包数。 | 用于判断红包玩法健康度，也影响返奖开奖资格。 |
| 风控拦截 | 当日因风险策略被拦截或不参与返奖的红包数。 | 命中黑名单、同设备/IP、多账号互抢、B 档聚集、预算熔断等策略的红包。 | 当前展示 `14`，用于运营快速感知风险规模。 |

## 4. 筛选区字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| 发送时间 | 时间范围 | 按红包发送成功时间筛选。默认展示当天 `00:00 - 23:59`。 |
| 红包 ID | 输入框 | 精确查询单个红包。用于排查投诉、补发、风控审核。 |
| 发送人 UID | 输入框 | 按红包发送人筛选。可用于定位某个用户的发红包行为。 |
| 房间 ID | 输入框 | 按红包所在房间筛选。可用于高风险房间排查。 |
| 红包类型 | 下拉 | 可选全部、金币红包、礼物红包。 |
| 返奖档位 | 下拉 | 可选全部、未达标、A 档、B 档。 |
| 红包状态 | 下拉 | 可选全部、待领取、已领完、过期部分领取、过期无人领取。 |
| 返奖状态 | 下拉 | 可选全部、未触发、未达到返奖条件、待开奖、已中奖、未中奖、风控拦截。 |
| 风控状态 | 下拉 | 可选全部、危险级：禁发禁领、限制级：不参与返奖、观察名单、正常。 |
| 重置 | 按钮 | 清空筛选条件并恢复默认查询口径。 |
| 查询 | 按钮 | 按当前筛选条件刷新列表和统计数据。 |

## 5. 红包数据列表字段

| 字段 | 含义 | 展示样式/示例 | 研发备注 |
| --- | --- | --- | --- |
| 红包 ID | 红包唯一标识，也是开奖唯一主键。 | `HB202604160012`，副文本可展示 `开奖主键`、`blocked_by_risk` 等辅助信息。 | 服务端返奖幂等必须以该字段为唯一开奖主键。 |
| 发送人 / 房间 | 红包发送人 UID 和发送房间 ID。 | 主文本 `UID 983211`，副文本 `房间 880321`。 | 风控跳转时需要携带 UID、房间 ID、红包 ID。 |
| 红包类型 | 区分金币红包和礼物红包。 | 金币红包为蓝色标签，礼物红包为紫色标签。 | 礼物红包金额按礼物价值折算金币。 |
| 金额 / 礼物 | 红包总金额或礼物总价值。 | `20,000 金币`、`66,640 金币`；副文本展示 A/B 档或礼物说明。 | 用于判断是否达返奖门槛。 |
| 领取配置 | 红包领取份数和发放方式。 | `10 份 / 随机`，副文本 `在房用户可领`。 | 包含份数、随机/均分、领取条件。 |
| 领取进度 | 当前已领取份数和状态。 | `10 / 10 已领完`、`18 / 20 已过期`。 | 领完后才可进入返奖开奖；未领完展示未触发。 |
| 返奖资格 | 当前红包命中的返奖档位和返奖规则。 | `A档：3%返5%`、`B档：5%返10%`。 | 未达标时应展示未达标或空状态。 |
| 返奖状态 | 当前红包返奖结算状态。 | `已中奖 / 返 1,000`、`未触发：未领完`、`未达到返奖条件`、`风控拦截`、`待人工复核`。 | 详情弹窗根据该字段判断展示完整返奖详情、领取情况或未中奖。 |
| 风控状态 | 当前红包命中的风控状态。 | `危险级：禁发禁领`、`限制级：不参与返奖`、`观察名单`、`正常`。 | 命中风控时通常不参与返奖，或进入人工复核。 |
| 预算影响 | 当前红包对今日返奖预算的影响。 | `+1,000 金币`、`0`、`冻结`。 | 已中奖扣减预算；待补发/待复核可预占或冻结预算。 |
| 发送时间 | 红包发送成功时间。 | `2026-04-16 10:08`。 | 建议后端返回秒级时间，前端按页面格式展示。 |
| 操作 | 当前红包可执行动作。 | `详情`、`领取记录`、`风控`。 | 详见第 6 节。 |

## 6. 列表操作说明

| 操作 | 交互 | 展示内容/结果 |
| --- | --- | --- |
| 详情 | 点击后打开红包详情弹窗。 | 弹窗包含“红包详情概览”和“选中返奖详情”两个模块。 |
| 领取记录 | 点击后打开领取记录列表弹窗。 | 展示该红包下的领取流水，点击流水行可查看单条领取详情。 |

| 刷新 | 点击后刷新当前查询结果。 | 不清空筛选条件。 |

### 7.2 领取情况（未达到返奖条件场景）

当选中红包未满足返奖条件时，详情弹窗上半部分不展示常规红包详情概览，而展示领取情况。典型场景包括红包未领完、过期未领完、有效领取不足、红包金额或礼物总价值未命中返奖档位。

| 字段 | 含义 |
| --- | --- |
| 红包 ID | 当前查看的红包唯一标识。 |
| 发送房间 | 红包发送所在房间 ID。 |
| 领取条件 | 在房用户、在麦用户等领取范围。 |
| 发放类型 | 随机或均分。 |
| 领取份数 | 红包总份数。 |
| 领取进度 | 当前已领取份数和完成状态。 |
| 返奖状态 | 当前未进入返奖开奖链路的原因状态。 |
| 风控状态 | 当前红包是否命中风控。 |
| 预算影响 | 未达到返奖条件时通常为 0。 |
| 发送时间 | 红包发送成功时间。 |
| 未达原因 | 说明为什么未进入返奖开奖链路，例如未领完、已过期、未达标等。 |

### 7.3 选中返奖详情

当列表中的返奖状态包含“已中奖”时，展示返奖详情。

| 字段 | 含义 |
| --- | --- |
| 返奖记录 ID | 当前红包对应的返奖记录唯一标识。 |
| 红包 ID | 关联红包 ID。 |
| 发送人 | 获得返奖的红包发送人 UID。 |
| 房间 | 红包发送房间 ID。 |
| 红包金额 | 当前红包总金币金额或礼物折算金币金额。 |
| 返奖档位 | A 档或 B 档。 |
| 返奖概率 | 当前档位配置的中奖概率。 |
| 返奖比例 | 当前档位配置的返奖比例。 |
| 返奖金额 | 本次实际返奖金币金额。 |
| 取整方式 | 返奖金额出现小数时的处理方式，当前展示向下取整。 |
| 钱包事务 | 返奖发放对应的钱包事务或补发状态说明。 |

当选中红包未满足返奖条件时，该模块只展示：

| 字段 | 含义 |
| --- | --- |

| 字段 | 含义 |

## 9. 页面下方明细模块

### 9.1 红包详情概览

该模块展示当前选中红包的静态概览，字段包括红包 ID、发送人、红包类型、发送房间、总金额、份数、领取条件、发放类型、倒计时、悬浮窗状态、返奖资格。

| 字段 | 含义 |
| --- | --- |
| 倒计时 | 红包领取有效期或剩余领取时间。 |
| 悬浮窗状态 | 房间内红包入口状态，例如已消失、领取中、已过期。 |
| 返奖资格 | 当前红包是否达标、命中的档位和本次返奖结果说明。 |

### 9.2 选中返奖详情

与详情弹窗中的“选中返奖详情”口径一致。用于在不打开弹窗时快速查看当前红包的返奖记录。

### 9.3 房间与风控指标

## 12. 测试验收点

- 已中奖红包点击详情，应展示红包详情概览和完整返奖详情。
- 未达到返奖条件的红包点击详情，上半部分应展示领取情况，下半部分只展示“未达到返奖条件”。
- 未中奖、风控拦截、待人工复核红包点击详情，选中返奖详情只展示“未中奖”。
- 点击领取记录，应展示当前红包下的领取流水；领取记录列表和单条领取详情均不展示领取结果和响应耗时。
- 点击风控，应跳转风控明细页并打开手动加黑弹窗，且带入红包 ID、发送人 UID、房间 ID。
- 今日返奖金额应与红包返奖记录页保持一致。
- 预算扣减明细中的剩余可用预算，应等于每日预算上限减去已发放和预占返奖金额。
