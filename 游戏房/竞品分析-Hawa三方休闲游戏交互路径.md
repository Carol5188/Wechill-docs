# 竞品交互路径分析 — Hawa 三方休闲游戏

> 版本：v1.0 | 日期：2026-05-20 | 来源：Hawa App 录屏（71s，iOS模拟器）  
> 分析范围：房间内调起游戏模式（Carrom Pool）完整交互路径

---

## 1. 竞品概览

| 项目 | 内容 |
|---|---|
| 竞品名称 | Hawa（هاوا） |
| 目标市场 | MENA（中东/北非），阿拉伯语为主 |
| 产品形态 | 语聊房 + 休闲游戏社交平台 |
| 游戏列表 | Jackaroo、UNO、Carrom Pool、PK Arena、Ludo、Catch Gift、Lucky Box、Lucky Dice、Lucky Number |
| 核心模式 | 语音聊天房内调起三方休闲游戏，游戏期间语音不中断 |

## 3. 按用户身份的交互流程图

### 3.1 房主（Room Owner）流程

房主 = 创建游戏房间的用户，拥有最高权限。

```mermaid
flowchart TD
    A[进入语聊房] --> B[点击右侧游戏手柄图标]
    B --> C[底部弹出 Mini Game 面板]
    C --> D{选择游戏}
    D -->|Carrom Pool| E[弹出游戏创建弹窗]
    D -->|PK Arena| E
    D -->|灰色不可用游戏| F[无响应/提示不可用]
    E --> G[配置参数]
    G --> G1[选择玩家数量: 2-4 players]
    G --> G2[选择匹配模式: Match players]
    G --> G3[是否自动创建下轮: ☑️/☐]
    G1 & G2 & G3 --> H[点击 Create]
    H --> I[进入游戏房间等待界面]
    I --> J[房主占1号位🏠]
    J --> K[空位显示 invite 按钮]
    K --> L[聊天区自动发送邀请消息]
    L --> M{等待玩家加入}
    M -->|人数不足| M
    M -->|人数达标| N[Start 按钮变为可用🟢]
    N --> O[点击 Start]
    O --> P[游戏加载 Loading 0%→100%]
    P --> Q[游戏对局进行中]
    Q --> R{对局结束}
    R --> S[WIN弹窗: 胜者Win+勋章 / 败者Out+😞]
    S --> T[倒计时 Return on Xs]
    T --> U[返回语聊房]
    
    style A fill:#2d5a3d,stroke:#4ade80,color:#fff
    style B fill:#1e3a5f,stroke:#60a5fa,color:#fff
    style E fill:#5a2d2d,stroke:#f87171,color:#fff
    style Q fill:#3d2d5a,stroke:#a78bfa,color:#fff
    style S fill:#5a4a1e,stroke:#fbbf24,color:#fff
    style U fill:#2d5a3d,stroke:#4ade80,color:#fff
```

**房主独有权限/操作：**
- 创建游戏（选择人数、匹配模式、自动创建选项）
- 点击 Start 开始游戏
- 结束游戏会话（系统提示："Room owners and administrators can now end game sessions!"）

### 3.2 普通玩家（Player）流程

普通玩家 = 通过邀请或匹配加入游戏房间的用户。

```mermaid
flowchart TD
    A[在语聊房内] --> B{收到游戏邀请}
    B -->|聊天区邀请消息| C[点击邀请/加入]
    B -->|房间内游戏入口| D[点击游戏手柄]
    D --> E[Mini Game面板]
    E --> F[选择对应游戏]
    C --> G[进入游戏房间等待界面]
    F --> G
    G --> H[占一个空位]
    H --> I[等待房主点击 Start]
    I --> J[游戏加载 Loading]
    J --> K[游戏对局进行中]
    K --> L{对局结束}
    L --> M[WIN弹窗]
    M --> M1{结果}
    M1 -->|胜| M2[Win + 勋章🏅]
    M1 -->|败| M3[Out + 😞]
    M2 & M3 --> N[倒计时 Return]
    N --> O[返回语聊房]
    
    style A fill:#2d5a3d,stroke:#4ade80,color:#fff
    style G fill:#1e3a5f,stroke:#60a5fa,color:#fff
    style K fill:#3d2d5a,stroke:#a78bfa,color:#fff
    style M fill:#5a4a1e,stroke:#fbbf24,color:#fff
    style O fill:#2d5a3d,stroke:#4ade80,color:#fff
```

**普通玩家限制：**
- 不可点击 Start（需等待房主）
- 不可修改游戏参数
- 可点击 invite 邀请其他用户

### 3.3 观战者（Spectator）流程

观战者 = 在语聊房内但未加入游戏对局的用户。

```mermaid
flowchart TD
    A[在语聊房内] --> B{游戏对局进行中}
    B --> C[观战者留在语聊房]
    C --> D[可看到麦位布局和聊天]
    C --> E[可正常语音聊天]
    C --> F[游戏画面不可见/不可操作]
    F --> G{对局结束}
    G --> H[可看到WIN弹窗结果]
    H --> I{是否加入下一轮}
    I -->|是| J[点击游戏手柄加入]
    I -->|否| K[继续留在语聊房]
    
    style A fill:#2d5a3d,stroke:#4ade80,color:#fff
    style C fill:#3d3d3d,stroke:#9ca3af,color:#fff
    style H fill:#5a4a1e,stroke:#fbbf24,color:#fff
    style J fill:#1e3a5f,stroke:#60a5fa,color:#fff
```

**观战者特征：**
- 语聊房功能不受影响（可上麦、聊天、送礼）
- 游戏进行期间无法看到游戏画面（全屏游戏覆盖）
- 对局结束后可看到结果
- 可随时通过游戏手柄加入下一轮

- 背景红色丝绒纹理
- 左侧显示用户头像+ID
- 底部仍可见聊天消息

## 5. 语音连续性设计

游戏全屏覆盖期间，语聊房**不中断**：

| 游戏阶段 | 语聊房状态 | 可见元素 |
|---|---|---|
| 游戏等待房间 | ✅ 持续 | 麦位、聊天区、礼物按钮 |
| 游戏加载中 | ✅ 持续 | 底部聊天消息、左侧用户头像 |
| 游戏对局中 | ✅ 持续 | （被游戏画面覆盖但语音不中断） |
| 结算弹窗 | ✅ 持续 | 背景可见麦位布局 |
| 返回语聊房 | ✅ 持续 | 完整恢复 |

## 6. 与WeChill PRD的关键差异对比

| 维度 | Hawa | WeChill PRD（当前） | 可借鉴点 |
|---|---|---|---|
| 游戏入口 | 右侧悬浮游戏手柄图标 | PRD有定义，需确认位置一致性 | ✅ 悬浮图标位置合理 |

| 创建弹窗参数 | 人数+匹配模式+自动创建下轮 | PRD缺少"自动创建下轮"选项 | ⚠️ 可考虑增加此选项 |
| 匹配模式 | Match players（系统匹配） | PRD有定义 | 基本一致 |
| 等待房间 | 房主+空位invite+灰色Start | PRD有定义 | 基本一致 |
| 游戏加载 | 全屏Logo+进度条 | PRD有定义 | 基本一致 |
| 结算弹窗 | WIN弹窗+倒计时自动返回 | PRD有定义 | ✅ 倒计时返回可参考 |

## 7. 可直接复用的设计建议

### 7.1 高优先级（建议采纳）

1. **自动创建下轮选项**：游戏创建弹窗增加"Whether to automatically create the game in the next round"复选框，减少房主重复操作
2. **结算倒计时自动返回**：WIN弹窗增加倒计时（5-7秒），避免用户卡在结算页
3. **创建成功自动邀请**：游戏创建成功后聊天区自动发送邀请消息"I created 🎯, come and join"

### 7.2 中优先级（建议评估）

4. **Explore页游戏匹配入口**：除房内入口外，增加Explore页直接匹配入口，降低使用门槛
5. **观战者结果可见**：对局结束后，观战者也可看到WIN弹窗结果，增强参与感

### 7.3 低优先级（备选参考）

6. **游戏列表3列网格布局**：比列表更紧凑，适合多游戏场景
7. **玩家数量下拉选择**：2-4人灵活配置，而非固定人数
