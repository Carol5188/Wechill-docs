WeChill Game Room · Client MVP v2.0

游戏房客户端页面原型

基于游戏房 MVP v2.0 文档和三方游戏截图，将 WeChill 语聊房壳、麦位、聊天、玩家座位与 Bytesun H5 游戏容器组合成可评审的前台页面。

MVP 边界
Ludo / UNO 优先 复用语聊麦位 不接排麦 不做随机匹配 不接玩法联动

1. 客户端主流程

MVP 先跑通“从游戏房入口进入、创建房间、加载 H5、上座开局、结算复玩”的闭环。随机匹配、画猜聊天同步、红包/龙蛋/CP 联动均后置。

MVP Flow
1

左侧页面为 WeChill 自研客户端页面；三方游戏页面作为 H5 容器内容嵌入。顶部房间信息、语聊麦位、底部聊天和礼物入口由 App 侧承接。

3. 客户端关键规则

这些规则来自 v2.0 MVP 文档，用来约束客户端实现和评审验收。

Implementation Rules
模块 页面表现 MVP 规则
语聊麦位 顶部横向麦位区展示上麦用户和空麦。 麦位和游戏座位分开；上麦不等于上游戏座位；不接排麦模式。
游戏座位 H5 内展示玩家位置，App 侧记录 GamePlayer。 上座 / 下座同步 `gameActionUpdate type=4`，全量座位同步 `type=16`。
观战 游戏中进入房间时展示观战态，可聊天和送礼。 以 `role=1` 进入，不参与 Ludo / UNO 局内操作。
声音 房间语音优先，游戏 BGM 可降低或关闭。 Ludo / UNO 继续使用我方语聊房 RTC，不启用游戏内 RTC。
结算 展示排名、得分、余额变化、再来一局。 `change_balance` 必须幂等；结算失败进入补偿和对账。
异常 加载失败、上座失败、维护、封禁、地区限制均有提示。 WebView 超时可重试；`type=18` 释放座位；维护游戏隐藏入口。
